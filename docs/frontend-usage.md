# 前端使用说明

本文面向普通用户、管理员和后续维护人员，说明 IDBS 前端页面的入口、功能和本地调试方式。前端为静态 H5 页面，代码主要位于 `public/`。

## 1. 页面入口

| 页面 | 访问路径 | 主要用途 | 登录要求 |
| --- | --- | --- | --- |
| 设备总览 | `/` 或 `/index.html` | 查看设备列表、状态概况、按关键词/分类/状态筛选设备 | 查看设备可公开；预约相关操作需用户登录 |
| 设备详情 | `/device.html?code=设备编号` | 查看设备基础信息、近 14 天占用、近期故障、当前使用和未来占用 | 查看可公开；预约/报备需用户登录 |
| 发起预约 | `/reserve.html` | 选择设备、日期和时间段，进行预约预检并提交预约 | 普通用户登录 |
| 使用日历 | `/calendar.html` | 按月/日查看设备使用安排 | 可公开展示，具体可见字段受后台配置影响 |
| 日历详情 | `/calendar-detail.html` | 查看某日或某条使用记录详情 | 可公开展示，具体可见字段受后台配置影响 |
| 我的记录 | `/my.html` | 查看预约、借还、通知、故障报备和需求上报；开始使用、提交归还 | 普通用户登录 |
| 聊天 | `/chat.html` | 与管理员或其他会话成员沟通，支持实时消息和图片 | 登录用户 |
| 用户登录 | `/login.html` | 账号密码登录、公众号验证码登录 | 无 |
| 微信注册/首次绑定 | `/register.html` | 说明首次绑定流程，引导从公众号验证码完成绑定 | 无 |
| 管理后台 | `/admin.html` | 后台工作台、设备、用户、预约、统计、系统配置、权限、日志等 | 管理员登录 |

> 生产部署后通常通过域名访问根路径，例如 `https://your-domain.com/`。如果系统配置了“关闭 IP 直连注册/浏览”，用户端公开页面也需要使用管理员配置的域名访问。

## 2. 普通用户流程

### 2.1 登录与首次绑定

1. 打开 `/login.html`。
2. 已有账号可使用手机号/密码登录。
3. 微信公众号登录时，在登录页生成验证码，并按页面提示发送到公众号。
4. 首次使用公众号验证码时，需要补充姓名、学号和手机号完成绑定。
5. 如果账号处于待审核、禁用或封禁状态，页面会给出对应提示，需要联系管理员处理。

### 2.2 查看设备和发起预约

1. 打开 `/index.html` 查看设备列表。
2. 使用搜索、分类和状态筛选快速定位设备。
3. 进入设备详情页查看位置、负责人、近期占用和故障摘要。
4. 点击预约入口进入 `/reserve.html`。
5. 选择一个或多个设备、日期和允许预约时间段。
6. 提交前系统会调用预约预检，检查账号状态、设备可用性、重复选择和时段冲突。
7. 预约提交后进入待审核或已通过状态，具体取决于后台规则和管理员审批。

### 2.3 借用、归还和通知

1. 打开 `/my.html` 查看“我的记录”。
2. 对已通过且到达使用条件的预约，点击开始使用。
3. 使用完成后提交归还；如果后台开启“归还必须上传图片”，需要先上传归还照片。
4. 异常归还、设备故障或管理员处理结果会进入通知区。
5. 通知支持全部标记已读。

### 2.4 故障/需求上报和聊天

- 在 `/my.html` 可提交故障报备或需求上报，并查看处理状态。
- 在设备详情或我的记录中可进入相关沟通入口。
- `/chat.html` 支持会话列表、消息发送、图片消息、未读状态、群信息抽屉和 SSE 实时推送；网络或浏览器限制下会自动降级为轮询。

## 3. 管理员流程

### 3.1 登录后台

1. 打开 `/admin.html`。
2. 使用管理员账号/密码登录。
3. 登录成功后可看到左侧后台模块导航。

### 3.2 常用后台模块

| 模块 | 功能 |
| --- | --- |
| 后台总览 | 查看 KPI、待处理事项和运行概况，可从指标跳转到对应筛选结果 |
| 数据分析 | 查看总览趋势、设备使用排行、时段热力和故障统计 |
| 新增设备 | 新建设备，维护分类、位置、负责人、图片、可预约时间段和可用状态 |
| 用户审核与微信绑定 | 审核用户、禁用/启用用户、封禁、解绑微信、查看用户详情 |
| 预约审核 | 查看预约批次摘要，展开后审核整批或单条预约，保留沟通入口 |
| 系统配置 | 修改登录提醒、预约可见字段、安全配置、公众号配置、日报推送等 |
| 使用统计与导出 | 查看使用记录，下载同步 CSV/Excel，或创建后台导出任务 |
| 注意事项 | 维护用户登录后显示的注意事项弹窗内容 |
| 用户信息 | 维护用户侧展示的系统通知类信息 |
| 工作人员联系方式 | 维护用户可见的管理员/工作人员联系方式 |
| 管理员与权限 | 维护角色、权限和管理员授权 |
| 故障报备 | 处理设备故障，必要时保持设备维修状态或恢复可用 |
| 需求上报 | 审核用户需求、变更申请和其他诉求 |
| 操作日志 | 查看后台关键操作记录 |

## 4. 前端代码结构

| 文件/目录 | 作用 |

| `public/*.html` | 页面结构，直接由 Express 静态服务输出 |

| `public/js/api.js` | REST 请求封装、token 存取、上传封装 |
| `public/js/ui.js` | 通用提示、toast、登录拦截和基础 UI 工具 |
| `public/js/common-header.js` | 公共导航、角色菜单、登录注意事项弹窗 |
| `public/js/reservation-slots.js` | 预约时间段选择和展示工具 |
| `public/js/calendar-ui.js` | 使用日历渲染逻辑 |
| `public/js/chat.js` | 聊天会话、消息、SSE/轮询和群信息抽屉逻辑 |
| `public/js/admin.js` | 管理后台主入口和模块编排 |
| `public/js/admin-workbench.js` | 后台工作台与 KPI |
| `public/js/admin-business.js` | 设备、预约、用户、故障和需求等业务管理 |
| `public/js/admin-system.js` | 系统配置、角色权限和基础资料 |
| `public/js/admin-ops.js` | 统计导出、操作日志和运维类后台能力 |
| `public/js/drawer.js` | 后台抽屉详情组件 |
| `public/js/page-shortcuts.js` | 页面快捷入口/辅助交互 |
| `public/css/style.css` | 全局样式 |

## 5. 本地调试

1. 安装依赖：

   ```bash
   npm install
   ```

2. 准备 `.env`。最小本地配置示例：

   ```bash
   PORT=3000
   ADMIN_PASSWORD=change-me-local
   TOKEN_SECRET=local-random-secret
   DATABASE_URL=postgresql://idbs_user:password@127.0.0.1:5432/idbs
   PGSSL=false
   CORS_ORIGIN=*
   ```

3. 启动服务：

   ```bash
   npm run dev
   ```

4. 浏览器打开：

   ```text
   http://127.0.0.1:3000/
   http://127.0.0.1:3000/admin.html
   ```

5. 如需连接独立后端，可在 `public/js/config.js` 或页面加载前覆盖：

   ```js
   window.APP_CONFIG = window.APP_CONFIG || {};
   window.APP_CONFIG.apiBaseUrl = 'http://127.0.0.1:3000';
   ```

## 6. 调试与验收建议

- 修改 HTML/JS/CSS 后执行 `npm run check`，检查语法、内联脚本和乱码。
- 后端服务启动后执行 `npm run smoke -- http://127.0.0.1:3000`，验证主要接口。
- 本地批量 E2E 或频繁刷新时，可临时提高限流，例如 `API_RATE_LIMIT_MAX=1000 npm run dev`。
- 如果浏览器看到旧样式或旧脚本，先强制刷新；必要时更新 HTML 中静态资源版本参数。
- 如果出现 `401`，先清理本地 token 并重新登录；如果管理员接口返回权限错误，检查后台角色授权。
