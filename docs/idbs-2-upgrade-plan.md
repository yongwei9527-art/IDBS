# IDBS 2.0 大升级方案

更新时间：2026-07-03

## 总目标

把系统升级为数据库干净、预约模型稳定、后端模块清晰、后台工作台化、用户端顺滑、页面柔和克制、部署可控的设备管理平台。

## 核心原则

- 不兼容旧数据库，采用全量升级。
- `sql/schema.sql` 是唯一数据库基线。
- 旧数据先备份，需要的再单独导入。
- 预约模型统一为 `reservation_batches + reservation_items`。
- 服务层按领域拆分，逐步减少巨型文件。
- 后台围绕待办、审批、设备、故障、统计展开。
- 用户端围绕快速预约、清楚记录、顺利归还优化。
- 删除废话、技术词、重复说明。
- 页面视觉柔和，切换自然，按钮反馈明确。
- 每个阶段都必须能检查、能验收、能回滚。

## M1：系统骨架升级

- 全量数据库基线。
- 预约模型统一。
- 借还/故障关联 `reservation_item_id`。
- 服务层初步拆分。
- `npm run check` 和 smoke 通过。

## M2：界面体验升级

- 删除无意义文字。

- 后台导航流畅切换。
- loading、toast、空状态统一。
- 移动端布局优化。

## M3：后台工作台升级

- 设备管理抽屉。
- 预约批次审批。
- 用户详情和角色授权。

## M4：沟通、统计、报表升级

## M5：生产可靠性升级

- 备份恢复文档。
