## 已完成范围

- 登录与入口：统一未登录跳转、用户端/后台端入口、管理员权限校验和基础安全配置。
- 用户端：设备列表、设备详情、预约前置校验、多设备多日期预约、日历视图、我的记录、通知已读、故障/需求统一上报、联系管理员聊天入口。
- 后台端：工作台指标、设备新增/编辑/停用/维修/恢复、设备历史详情、预约批次审核、用户审核/禁用/解绑、用户详情、故障处理、用户诉求处理、操作日志、权限角色管理。
- 聊天：用户端和后台端会话、SSE 实时推送、轮询降级、未读状态、上下文卡片、图片消息、聊天用户/会话接口；普通群支持创建者解散、成员主动退出，实验管理总群禁止退出和解散；群信息改为右上角按钮触发的悬浮抽屉，包含群公告、成员列表和群管理，避免默认占用聊天主视图区。
- 数据库：预约批次/明细、设备时间段、权限、操作日志、聊天、上下文卡片、实时基础表等迁移脚本已补齐。
- API：已统一为 REST 路由，旧版 `POST /api/:action` 兼容映射已删除，新增用户详情、设备详情、统计导出、故障关闭/保留维修状态等契约。
- 导出：统计页支持 CSV 和 Excel 导出，覆盖使用记录、预约、故障、用户活动、设备汇总。
- 验收：`npm run check`、`npm run smoke` 覆盖基础健康检查、鉴权、预约预检、后台指标、后台详情、导出和聊天 SSE。
- 本轮复测：`npm.cmd run check` 通过；在当前工作区服务 `http://127.0.0.1:3012` 上 `npm.cmd run smoke` 全部通过。3000 端口曾命中旧服务，后续本地验证建议先确认端口进程或使用独立端口。
- IDBS 2.0 体验升级：后台用户详情、设备历史、预约明细、故障详情和需求详情已优先使用抽屉展示，减少跳页和整页刷新。
- 预约审核 UI：预约批次卡片默认只显示紧凑摘要，展开后再加载 `/admin/reservation-batches/:id` 明细；整批/单条审批按钮仅在展开后显示，沟通入口保留，审批后保持当前展开上下文。
- 预约模型收口：日历、后台批次列表、通知详情、统计视图、设备下一预约快照、设备故障通知、借还记录和故障报备均以 `reservation_items` 为事实来源；`/bookings` 保留为兼容别名，旧 `reservations` 仅作兼容影子更新/历史回填。
- 交互反馈：关键按钮已接入 loading 状态，工作台 KPI 支持带筛选跳转，故障/需求/预约详情保留局部刷新和上下文沟通入口。
- E2E：已新增 Playwright 配置和端到端用例骨架，覆盖后台模块、友好文案、抽屉组件、预约/故障/需求详情展示、移动端基础布局。
- E2E 当前状态：Playwright Chromium 依赖已可用；在 `http://127.0.0.1:3012` 上以当前 Chromium 路径运行 `npm.cmd run e2e`，结果为 33 passed、1 skipped。为避免本地 E2E 批量资源请求触发 429，测试服务建议设置 `API_RATE_LIMIT_MAX=1000`。
